# Tello Talent Planning Adaptation Requirements

## Scope
Adapt the existing SETS planning workflow for DJI Tello Talent scale flight while keeping [scripts/policy_convergence.py](scripts/policy_convergence.py) as the execution entry point.

## Hard Constraints
1. Do not modify [scripts/rollout.py](scripts/rollout.py).
2. Configuration changes should be minimal and primarily YAML-driven.
3. Start and goal distance must be at least 30 m.
4. Two obstacles must lie on the start-goal line.
5. Final visualization must be generated through [scripts/plot_rollout_data.py](scripts/plot_rollout_data.py).

## Parameterization Requirements
1. Use dedicated staged YAML files under [configs/sixdofaircraft](configs/sixdofaircraft):
   - [configs/sixdofaircraft/policy_convergence_tello_stage1.yaml](configs/sixdofaircraft/policy_convergence_tello_stage1.yaml)
   - [configs/sixdofaircraft/policy_convergence_tello_stage2.yaml](configs/sixdofaircraft/policy_convergence_tello_stage2.yaml)
   - [configs/sixdofaircraft/policy_convergence_tello_stage3.yaml](configs/sixdofaircraft/policy_convergence_tello_stage3.yaml)
2. Keep existing closed-loop planning structure (UCT-MPC through policy convergence flow).
3. Use progressive shrink toward Tello-scale rigid body and control bounds:
   - Stage-1: stable baseline envelope
   - Stage-2: intermediate shrink
   - Stage-3: near-real Tello envelope with feasibility margin
4. Preserve compatibility with existing output pipeline in [data](data) and [plots](plots).
5. Keep [scripts/policy_convergence.py](scripts/policy_convergence.py) as the only execution entry and switch stages via environment variable (`TELLO_STAGE`).

## Verification Requirements
1. YAML syntax loads successfully.
2. Geometric constraints validation:
   - Euclidean distance between start and goal >= 30 m.
   - Both obstacle centers collinear with start-goal segment.
3. [scripts/policy_convergence.py](scripts/policy_convergence.py) runs successfully with the new config and generates rollout result files.
   - `TELLO_STAGE=stage1`
   - `TELLO_STAGE=stage2`
   - `TELLO_STAGE=stage3`
4. [scripts/plot_rollout_data.py](scripts/plot_rollout_data.py) consumes generated data and outputs a PDF figure set.

## Verification Record
1. Geometry constraints passed for all three stages:
   - Start-goal distance = 40.0 m
   - Two obstacles on the start-goal line (center-to-line error = 0)
2. Runtime entry validation passed for all three stages through [scripts/policy_convergence.py](scripts/policy_convergence.py).
3. Final visualization generated by [scripts/plot_rollout_data.py](scripts/plot_rollout_data.py):
   - [plots/tello_rollout_three_stage.pdf](plots/tello_rollout_three_stage.pdf)
